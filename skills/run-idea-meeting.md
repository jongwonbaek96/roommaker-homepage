# 스킬: 사업 아이디어 회의 진행

## 언제 쓰나
- "아이디어 회의 하자", "이 아이디어 어때?", "새로운 게임 기획 얘기 나눠보자"
- 아이디어가 있긴 한데 어디서 시작해야 할지 모를 때

## 절차

### Phase 1: 회의 시작 (5분)
1. MEMORY.md 확인 — 이전에 비슷한 아이디어 논의했는지 체크
2. 오늘 회의 안건 확인: "오늘 구체적으로 어떤 아이디어를 얘기할까요?"
3. 목표 설정: "이 회의가 끝날 때 무엇을 결정하고 싶으세요?"

### Phase 2: 아이디어 탐색 (15~20분)
다음 순서로 질문하며 아이디어를 구조화:

```
1. 한 줄 요약: "이 아이디어를 한 문장으로 설명하면?"
2. 타깃: "누구를 위한 것인가? 어떤 사람이 즐길까?"
3. 핵심 경험: "플레이어/유저가 뭘 느껴야 성공인가?"
4. 차별점: "비슷한 게임/서비스 대비 왜 이걸 선택하나?"
5. 개발 난이도: "혼자/소팀으로 만들 수 있는 규모인가?"
```

### Phase 3: 비판적 검토 (10분)
`validate-idea.md` 스킬 적용:
- Kill Criteria 3가지 도출 (이 조건이면 만들지 않는다)
- 가장 큰 리스크 1가지 식별
- 빠른 검증 방법 제안

### Phase 4: 회의 정리 (5분)
1. 결정 사항 요약
2. 다음 단계 3가지 이내로 정리
3. `write-meeting-notes.md` 스킬로 회의록 작성
4. MEMORY.md에 핵심 결정 추가

## 체크리스트
- [ ] 회의 시작 전 MEMORY.md 확인했는가
- [ ] 아이디어 한 줄 요약이 나왔는가
- [ ] 타깃 유저가 명확해졌는가
- [ ] Kill Criteria가 정의됐는가
- [ ] 다음 액션이 결정됐는가
- [ ] 회의록이 작성됐는가

## 주의사항
- 아이디어가 "좋다/나쁘다" 이분법으로 끝나면 실패한 회의
- 다음 단계가 "더 생각해보기"이면 실패한 회의 — 구체적인 액션으로 쪼개라
- 회의 시간이 1시간을 넘으면 안건이 너무 많은 것 → 다음 회의로 분리
