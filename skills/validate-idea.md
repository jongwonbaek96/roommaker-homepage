# 스킬: 아이디어 검증

## 언제 쓰나
- 아이디어가 구체화됐을 때 실행 가치가 있는지 판단
- 여러 아이디어 중 우선순위를 정해야 할 때
- "이거 만들 가치 있어?" 질문이 나올 때

## 절차

### 1단계: 린 캔버스 빠른 버전 (10분)

| 항목 | 질문 |
|------|------|
| **문제** | 어떤 구체적인 불편함/욕구를 해결하나? |
| **솔루션** | 핵심 기능 3가지 |
| **UVP** | 한 문장 가치 제안 |
| **경쟁 우위** | 복사하기 어려운 차별점 |
| **고객 세그먼트** | 얼리어답터는 누구인가? |
| **수익 모델** | 어떻게 돈 버나? |
| **비용 구조** | 주요 비용 항목 |

### 2단계: RICE 점수 계산

```
Reach (도달 범위): 월 몇 명이 쓸 수 있나? (1~10점)
Impact (임팩트): 유저 1명에게 얼마나 큰 변화? (0.25/0.5/1/2/3)
Confidence (확신도): 이 추정의 신뢰도? (%)
Effort (노력): 개발에 몇 달 필요? (인월 단위)

RICE Score = (Reach × Impact × Confidence) / Effort
```

### 3단계: Kill Criteria 도출

아이디어를 폐기할 조건을 미리 정의:
```
이 아이디어는 다음 조건이면 진행하지 않는다:
1. [기술적 조건] — 예: 솔로 개발로 6개월 내 불가능하면
2. [시장 조건] — 예: 유사 게임이 이미 3개 이상 서비스 중이면
3. [수익 조건] — 예: 무료 플레이 모델로 수익화 경로가 없으면
```

### 4단계: 최소 검증 실험 설계

"이 아이디어가 유효한지 가장 빠르게 확인하는 방법은?"

```
실험 유형 선택:
- 페이퍼 프로토타입 — 코딩 없이 UI/흐름 검증
- 랜딩 페이지 테스트 — 관심도 측정
- 코어 루프 프로토타입 — 재미 여부만 검증
- 유저 인터뷰 5명 — 문제 유효성 확인
```

## 판정 기준

| RICE 점수 | 판정 |
|-----------|------|
| 상위 20% | 즉시 진행 |
| 중간 60% | 검증 후 결정 |
| 하위 20% | 보류 또는 폐기 |

## 체크리스트
- [ ] 린 캔버스 7항목 채웠는가
- [ ] RICE 점수 계산했는가
- [ ] Kill Criteria 3개 정의됐는가
- [ ] 최소 검증 실험이 설계됐는가

## 주의사항
- RICE 점수는 절대값이 아닌 상대 비교용
- Kill Criteria는 감정 개입 없이 사전에 정의해야 효과가 있음
- "일단 만들어보자"는 검증 없는 실행 — Kill Criteria 없으면 피벗 타이밍을 놓친다
