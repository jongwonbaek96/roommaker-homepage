## 새 스킬 작성 규칙

파일명: `{동사}-{목적어}.md` (예: `write-gdd.md`, `analyze-competitor.md`)

```markdown
# 스킬명

## 언제 쓰나
이 스킬을 트리거하는 상황 설명

## 절차
1. 첫 번째 단계
2. 두 번째 단계
...

## 체크리스트
- [ ] 확인 항목 1
- [ ] 확인 항목 2

## 주의사항
- 실패 패턴이나 피해야 할 것들

## 레퍼런스
- 참고한 예시나 링크
```

## 스킬 생성 트리거

다음 상황에서 스킬 파일을 만든다:
- 같은 유형의 작업을 두 번 이상 했을 때
- 복잡한 작업을 마치고 "다음에도 이렇게 하면 되겠다"는 판단이 들 때
- 피그말리온이 "이거 정리해둬"라고 요청할 때
